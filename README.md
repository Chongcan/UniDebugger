# UniDebugger 无人车异常行为排查工具箱
## 项目简介
UniDebugger是一套面向校园无人车工程师的轻量化异常行为排查工具箱，针对校园场景下无人车异常难还原、线索分散、判定不严谨、展示不直观四大痛点，以"数据自动捕获—多维线索提取—候选原因推理—可视化呈现"四阶段流水线为核心，辅助工程师快速还原异常现场、形成排查方向，适配校园末端配送无人车研发测试场景。

## 项目背景
2026年《智能网联汽车道路测试与示范应用安全通行规范》正式施行，无人车黑匣子数据强制留存，行业从数据匮乏进入数据洪流阶段，但缺少轻量化的日常排查工具。校园作为末端配送核心场景，异常行为频发，传统人工跨日志排查效率极低。

## 整体架构
本项目分为四大核心模块：
1. 异常场景自动捕获模块：基于规则扫描黑匣子数据，自动截取异常事件前后10秒全量数据快照
2. 关键线索多维提取模块：提取空间/车辆/感知/决策四类标准化线索，覆盖环境端与车辆端全链路
3. 候选原因智能判断模块：规则初筛+大模型辅助，输出带置信度的候选原因与排查建议，不做最终定论
4. 可视化协作模块：时间线+多维度数据曲线+地图标注的可视化面板，支持团队协作复盘

## 技术栈
- 数据处理：Python、Pandas、ROS Bag解析
- 可视化：Matplotlib、Streamlit
- 协作载体：飞书开放平台、多维表格、智能伙伴
